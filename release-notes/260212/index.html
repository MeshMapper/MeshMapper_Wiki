    üöÄ MeshMapper Update - Admin Tools & Map Enhancements (February 12, 2026)¬∂

    ‚ú® New Features¬∂

    • Detailed vs. Simplified Map Views ‚Äî Added a toggle in Settings to switch between "Detailed" (full grid resolution) and "Simplified" (clustered repeaters, optimized grid) modes. Simplified mode significantly improves performance on slower devices. Simplified mode is now the default.
    • Discord Integration ‚Äî Region administrators can now link their Discord! This will allow administrators to receive notifications about their region from the MeshMapper bot (for example, when a new repeater has sent an advert and its ID is now in collision). Find this setting in the "User Settings" tab of the admin panel.
    • Administration Popup ‚Äî A new "Administration" option on the map displays the region's administrators and contact information, making it easier for visitors to reach out to local maintainers. Visitors can send messages to the administrators through the Discord integration/MeshMapper bot here, if there is at least one region administrator with their Discord linked and that notification type is enabled.
    • Region Message ‚Äî Administrators can now set a custom message (e.g., local Discord server invite, website link) that appears to visitors in the Administration popup. This allows administrators to direct map visitors to local resources.
    • Disable Duplicate ID Detection ‚Äî Added the ability for region admins to opt-out of strict duplicate ID detection. This allows regions to visualize all data even if repeater IDs collide. Read about this in more detail here.
    • Onboarding Status ‚Äî New regions pending onboarding will display in the Maps dropdown with a landing page indicating that onboarding is pending. Optionally linking ones Discord with the onboarding form allows for automated status updates sent directly from the MeshMapper bot.
    • Users ‚Äî Region administrators now have the ability to create Users. Creating a User allows you the ability to combine multiple companions under a single "primary" companion, which will aggregate a users statistics and contributions under that single companion on the leaderboard. This is also the first stepping stone into a planned portal that will allow end-users the ability to manage their own contributions, see their own detailed statistics, etc.
